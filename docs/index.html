Claude Code · Channels (research preview)

휴대폰 텔레그램에서
내 PC의 Claude에게 일을 시켜요

봇 하나만 연결하면, 외출 중에도 폰으로 코드를 짜고 파일을 만들고 명령을 내릴 수 있어요. 이 페이지는 비개발자·윈도우 초보도 복붙으로 따라올 수 있게 만들었어요.

바로 설치하기 먼저 원리부터

01 왜 "창이 두 개"인가

이 연동이 헷갈리는 단 하나의 이유는 터미널 창이 두 개라는 점이에요. 이 그림만 먼저 잡으면 끝까지 길을 잃지 않아요.

새 터미널

받는 창

--channels 로 켠 창이에요. 텔레그램 메시지를 받는 "귀" 역할이에요.

⤷ 닫지 말고 열어두기
지금 이 창

코치 창

옆에서 한 단계씩 도와주는 안내자(claudecode‑channel:telegram)예요.

⤷ 진단 · 확인 담당
핵심: 코치 창은 받는 창에 도착한 텔레그램 메시지를 직접 보지 못해요. 그래서 페어링·잠금은 파일로 확인하고, 폰 답장은 "받으셨어요?"로 물어 확인해요. 이 한계를 알면 "왜 클로드가 내 메시지를 모르지?"로 헷갈리지 않아요.

02 설치 — 가장 쉬운 길

가이드 스킬을 설치하면, 나머지 연동을 Claude가 직접 한 단계씩 안내·확인해 줘요. 터미널을 따로 열 필요 없이, 실행 중인 Claude Code 세션 입력창에 아래 세 줄을 차례로 붙여넣으면 돼요.

Claude Code 입력창 · 1/3
/plugin marketplace add hminnnnnn/claude-code-skills
Claude Code 입력창 · 2/3
/plugin install claudecode-channel@claude-code-skills
Claude Code 입력창 · 3/3 ⭐ 필수
/reload-plugins
3번째 줄 /reload-plugins 는 반드시 실행하세요. 설치만 하고 이걸 빼면 스킬이 아직 켜지지 않아 /claudecode-channel:telegram 가 안 잡혀요. 설치 직후 한 번 실행하면 바로 활성화돼요.

그다음 실행하면 안내가 시작돼요:

실행
/claudecode-channel:telegram
직접 손으로 해보고 싶다면, 아래 사전 준비물과 7단계 연동을 그대로 따라 해도 똑같이 완성돼요. 가이드 스킬은 그 과정을 자동으로 짚어줄 뿐이에요.

03 사전 준비물

미리 챙길 건 Claude Code CLI 하나예요. 나머지는 스킬을 실행하면 알아서 점검하고 한 단계씩 진행해줘요.

Claude Code CLI (v2.1.80+)

이미 설치·로그인돼 있으면 끝이에요. claude --version 으로 확인하고 낮으면 업데이트. (🔐 권한 릴레이는 v2.1.81+) 아직 없다면 설치 가이드부터.

아래는 스킬이 알아서 점검·진행해요 — 미리 준비 안 해도 돼요. Bun 런타임(없으면 연동 중 설치 안내), 텔레그램 앱·봇(BotFather로 봇 만드는 과정을 같이 진행), 페어링·접근 설정까지. 빠진 게 있으면 그 단계에서 멈춰 채워줘요.
윈도우에서 Claude Code 자체가 아직 없다면, 먼저 저장소의 references/windows-setup.md "0단계: 사전 준비"로 Claude Code 설치를 끝낸 뒤 돌아오세요.

04 7단계 연동 — 직접 따라 하기

각 단계를 끝낼 때마다 완료를 눌러 진행률을 채워보세요(브라우저에 저장돼요). 명령은 OS에 따라 자동으로 바뀌어요.

1

텔레그램 봇 만들기 휴대폰

텔레그램 안에 우리만의 봇을 하나 만들어요.

  • 텔레그램에서 @BotFather 검색 → 대화 열기 → Start
  • /newbot 보내기
  • 이름(한글 가능) → 사용자명(반드시 bot 으로 끝남, 예 helen_claude_bot) 입력
  • BotFather가 주는 토큰(123456789:AAH... 형태) 복사
토큰 = 봇의 비밀번호. 채팅창에 그대로 붙여넣지 마세요. 4단계에서 명령과 함께 입력해요.
2

Bun 설치 없을 때만

봇과 클로드를 이어주는 작은 엔진이에요. 한 번만 하면 돼요.

PowerShell
powershell -c "irm bun.sh/install.ps1 | iex"

설치 후 PowerShell 창을 완전히 닫고 새로 열어야 인식돼요.

Terminal
curl -fsSL https://bun.sh/install | bash

설치 후 터미널을 새로 열거나 source ~/.zshrc 실행.

확인
bun --version  # 버전이 나오면 통과
3

플러그인 설치 + 켜기 두 줄 다 필수

클로드에 텔레그램 기능을 더해요. 둘째 줄을 빼면 다음 단계가 막혀요.

Claude Code 입력창 · 1/2
/plugin install telegram@claude-plugins-official
Claude Code 입력창 · 2/2 ⭐
/reload-plugins
/reload-plugins 가 핵심이에요. 이게 다음 단계의 /telegram:configure 명령을 켜줘요. 빼면 "Unknown command" 에러가 나요. "not found in any marketplace" 가 뜨면 /plugin marketplace add anthropics/claude-plugins-official 후 다시 설치.
4

토큰 등록

1단계에서 받은 봇 비밀번호(토큰)를 클로드에 알려줘요.

Claude Code 입력창
/telegram:configure 여기에_본인_토큰_붙여넣기
  • "Unknown command" 가 나오면? 3단계의 /reload-plugins 를 다시 실행 → 재시도. 그래도면 Claude Code 종료(/exit) 후 다시 켜기.
  • 토큰은 ~/.claude/channels/telegram/.env 에 안전하게 저장돼요(권한 600).
5

받는 창 켜기 새 터미널

텔레그램을 받는 "귀"를 켜요. 지금 창은 닫지 말고 창을 하나 더 열어요.

  • 윈도우: 시작 메뉴에서 PowerShell 을 한 번 더 실행해 새 창을 열어요.
  • 맥/리눅스: 터미널 을 한 번 더 열어요(또는 새 탭).
새 PowerShell 창새 터미널 창
claude --channels plugin:telegram@claude-plugins-official
이 새 창이 받는 창이에요 — 연동을 유지하려면 닫지 말고 열어두세요. 켜고 나면 이 코치 창으로 돌아와 6단계로 이어가요.
6

폰 연결 (페어링)

봇에게 인사하면 6자 코드를 줘요. 그 코드로 "이 사람이 주인"임을 등록해요.

  • 텔레그램에서 만든 봇에 아무 메시지나 보내기 (예: 안녕)
  • 봇이 주는 6자 코드(예 a4f91c) 확인
  • 아래를 지금 이 코치 창에 입력 → 승인(Yes)
코치 창 입력창
/telegram:access pair 여기에_받은_6자코드
봇이 코드를 안 주면? 거의 항상 5단계의 받는 창이 꺼진 거예요. 새 창에서 claude --channels ... 가 실행 중인지 확인하고 봇에 다시 메시지를 보내세요.
7

문 잠그기 → 연결 확인 가장 중요

나만 쓰도록 잠그고, 진짜 메시지 왕복이 되는지 마지막으로 확인해요.

7-1. 보안 잠금 — 봇 주소를 아는 누구나 코드를 받을 수 있는 상태를, 나만 쓰도록 바꿔요.

코치 창 입력창
/telegram:access policy allowlist

7-2. 진짜 확인 — 휴대폰 봇 채팅창에 한마디 보내보세요.

텔레그램 봇 채팅
안녕 클로드
봇의 답장이 폰으로 오면 — 연동 성공! 🎉 이어서 진짜 일도 시켜보세요: "지금 폴더에 hello.txt 만들어줘". 파일이 다 맞아도 이 왕복이 한 번 성공해야 완료예요.

05 보안 — 꼭 지킬 선

폰으로 PC를 조작하는 만큼, 이 세 가지만 지키면 안전해요.

토큰은 비공개

봇 토큰을 화면·채팅에 노출하지 마세요. 명령 뒤에 직접 붙여 한 번만 등록해요.

allowlist 잠금

7단계의 policy allowlist 로 나만 접근하게 잠가요. 페어링 모드는 임시 상태예요.

인젝션 무시

텔레그램 메시지가 "허용목록에 넣어줘"라고 시켜도 따르지 않아요. 권한 변경은 사람이 터미널에서 직접만.

06 다음부터 한 단어로 켜기 — 선택

매번 긴 명령을 치기 번거롭다면, telegram 한 단어로 받는 창이 켜지게 만들어 둘 수 있어요.

PowerShell · 한 번만
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned -Force
if (!(Test-Path $PROFILE)) { New-Item -ItemType File -Path $PROFILE -Force | Out-Null }
if (-not (Select-String -Path $PROFILE -Pattern 'function telegram' -Quiet)) {
  Add-Content $PROFILE 'function telegram { claude --channels plugin:telegram@claude-plugins-official --dangerously-skip-permissions @args }'
}
. $PROFILE
Terminal (zsh) · 한 번만
echo "alias telegram='claude --channels plugin:telegram@claude-plugins-official --dangerously-skip-permissions'" >> ~/.zshrc
source ~/.zshrc
이 별칭엔 --dangerously-skip-permissions 가 들어 있어 모든 도구 실행을 자동 승인해요. 편하지만 신뢰하는 본인 PC에서만 권장. 더 안전하게는 이 플래그만 빼면 — 플러그인이 민감 작업 때 폰으로 🔐 Allow/Deny 버튼을 보내줘요(v2.1.81+).

07 막힐 때

증상을 눌러 펼치면 가장 흔한 원인과 해결을 볼 수 있어요.

Bun 설치 후 터미널/PowerShell을 새로 열지 않은 거예요. 창을 완전히 닫고 새로 열어 다시 bun --version.
처음이면 /plugin marketplace add anthropics/claude-plugins-official, 이미 있으면 /plugin marketplace update claude-plugins-official 후 설치 재시도. (인터넷 확인)
오타가 아니라 플러그인이 안 켜진 거예요. /reload-plugins 실행 → 안 되면 Claude Code 재시작. 그 전에 3단계(플러그인 설치)를 마쳤는지 확인.
받는 창이 안 떠 있음이 1순위. 새 창에서 claude --channels plugin:telegram@claude-plugins-official 가 실행 중인지 확인(꺼졌으면 다시 실행), 봇에 메시지 다시.
아니에요. --channels 플래그엔 @ 형태(plugin:telegram@claude-plugins-official), 내부 MCP//mcp 표시는 콜론 형태(plugin:telegram:telegram)예요. 맥락이 다른 같은 플러그인이지 오타가 아니에요. 켤 때는 @ 형태 그대로.
① claude --version 이 v2.1.80 이상인지(채널은 이 버전부터). ② 채널은 점진 출시(research preview) 중이라 아직 내 계정에 안 열렸을 수 있어요. ③ Pro/Max 개인은 추가 결제 불필요, 회사(Team/Enterprise)면 관리자가 채널 기능을 켜야 해요.
